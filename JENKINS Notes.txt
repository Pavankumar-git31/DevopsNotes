CRON JOB : used to schedule the works on Jenkins
cron syntax is used to work cron jobs
cron syntax having * * * * *
each star will have the spaces in between them 

*	: represents minutes 
*	: hours 
*	: date 
* 	: month
*	: day (sun=0 ,1=mon)

i want the build for 11:05 08-09-2025
5 11 8 9 1 --> this is the cron syntax 

4:45 pm 09-09-2023
45 16 9 9 6

go to Jenkins --> configure --> build periodically --> add syntax and save 
Linux will follow the UTC time zone not Indian time zone 

Limitations:
will not check the code is changed or not 

POLLSCM: it will build when the code is changed only 
Note : no need to set the time limit 

Create a CI job : Build Triggers --> Poll Scm --> * * * * * --> save 

Limitation:
you want build at 9 am . but developer changed the code at 7 am only --> time wastage is there 

inorder to remove this time wastage we are going to use WEBHOOKS.

WebHooks:
it will trigger the build at the moment we commit the code  

git hub --> repo --> settings --> webhooks --> add webhook
PayloadUrl : Jenkins server ip_adress:8080/github-webhook/
content type : application/json --> add webhook

jenkinsDashboard --> create CI job --> configure --> Build triggers --> Git hub hook triggers --> save

JENKINS MASTER AND SLAVE :
when we click on the multiple builds on the master node the Jenkins server might be crash. in real time we have to the multiple builds so the in order to avoid on the load on the master node we created on slave node to helps to take that build load on master. so nothing but master is the main node the slave which took the builds to do not crash the master server.

How to create the slave node :
create a ec2 instance and install the maven and java version -8 . because in order to build the project we need the maven. after that. we have to install the java 11. because agent slave is the agent to do master builds. so agent supports the java11 only.
after installing the java 11. 
go to Jenkins master dashboard --> Manage Jenkins --> nodes --> built in node --> new node --> give the name for the new node --> number of executors (as you need ) --> remote/root directory /tmp --> host --> use private ip address salve1 for kind select the username with keypair  and enter user name and click on enter directly --> add the key pair and save and --> add the key pair --> select the option build only when label matches the node --> select the last options for all the drop downs --> enter label name --> save.

automatically when you click on the build the war file is going to store in the /tmp folder in slave server.

PIPELINE:
step by step execution of the particular process
or 
series of events interlinked with each other 
code --> build --> test --> deploy 
pipe lines are syntax based
it is domain specific language(DLS)
we use groovy language to write the pipeline 
it gives the visualization of a process execution (free style will not give visualization)
TYPES OF PIPELINE:
1.SCRIPTED
2.DECLARATIVE(high priority) --> it gives a lot of flexibility

DECLARATIVE(in real time mostly used) :
to write the pipelines : to create a pipeline first 
go to Jenkins dashboard --> new item --> pipeline --> write the script 
pipeline{
     agent any
	stages{
	   stage('one){
	     steps{
		sh 'touch file1'
		}
	     }
	}
    }


if you want to give the clean package command we have to first integrate with the git for that 
we have to select the pipeline syntax 
click on pipeline syntax --> sample step --> git --> paste the repo link and --> click on generate pipeline script.

PIPELINE AS A CODE :
executing the multiple commands and multiple actions on a single stage --> write the all the code in single stage 

PIPELINE AS CODE AS A SINGLE SHELL:
sh '''
mvn compile
mvn build 
mvn test
mvn clean package 
'''
