BLOCK STORAGE TYPES:
EBS :
-> persistence: if you stop or reboot the server the data will not be deleted.
-> if you terminate it will delete.
-> N/W attached to virtual drive.
-> N/W attached is slow than physical attached.
-> maximum limitation is 16TB of storage.

INSTANCE STORE BACKED EC2 :
-> the virtual hard drive on the host is allocated to ec2 instances.
-> it is limited to some instances.
-> limited to 10GB per device.
-> Non-persistence ( Ephermeral)
-> can only be rebooted if you stop or terminate the data will be deleted.


EBS :
-> it's a raw Un-formatted , external block storage that you can attach to EC2 instances.
-> EBS Attached to EC2 through the AWS network , like virtual hard drive.
-> Multiple EBS volumes can be attached to single EC2 instance.
-> at a time we can attach the single EBS volume to EC2 instance.
-> EBS volumes and instances must be in same availability Zones.
-> the Data in EBS volumes will be replicated in multiple copies in same AZ to prevent data      
   data loss.
